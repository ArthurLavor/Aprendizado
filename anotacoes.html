Nesse site, irei anotar tudo que faço, criarei listas de tarefas e o que me der vontade

Commits por tipo de alteração

  • git init
    Exemplo:
    Inicia um novo projeto com git

  • git add
    Exemplo:
    add os arquivos que estão prontos para serem commitados

  • git commit -m
    Exemplo:
    commit os arquivos do histórico

  • git log
    Exemplo:
    mostra os últimos commits realizados

  • git status
    Exemplo:
    mostra como está o status da nossa ramificação

  • git diff
    Exemplo:
    mostra o que foi alterado / o que tem de alteração na ramificação

  • git merge
    Exemplo:
    merge de ramificações, mescla as alterações de uma ramificação na outra

  • git checkout
    Exemplo:
    muda para essa branch

  • git branch -b
    Exemplo:
    muda para essa branch e cria uma nova branch a partir da atual que estamos

  • git remote add
    Exemplo:
    adiciona um repositório remoto, ou seja, adiciona um repositório que está hospedado em outro lugar (GitHub, GitLab, etc.)

  • git push
    Exemplo:
    manda as nossas alterações para o repositório remoto

  • git pull
    Exemplo:
    pega as alterações do repositório remoto e traz para o nosso repositório local

  • git fecth
    Exemplo:
    atualiza o novo histórico do repositório remoto, mas não faz o merge com o nosso repositório local

  • feat:
    Adição de uma nova funcionalidade. Exemplo:
    feat: implementar sistema de login

  • fix:
    Correção de bugs ou problemas existentes. Exemplo:
    fix: corrigir erro de autenticação

  • refactor:
    Alterações no código que não modificam o comportamento, mas melhoram a estrutura ou legibilidade. Exemplo:
    refactor: reorganizar lógica de autenticação

  • style:
    Alterações relacionadas à formatação, como indentação ou remoção de espaços, sem impacto funcional. Exemplo:
    style: ajustar espaçamento em arquivo HTML

  • test:
    Adição ou alteração de testes automatizados. Exemplo:
    test: criar testes para função de login

  • docs:
    Alterações na documentação. Exemplo:
    docs: atualizar instruções no README.md

  • chore:
    Tarefas de manutenção, como atualização de dependências ou configurações. Exemplo:
    chore: atualizar versão do Node.js
